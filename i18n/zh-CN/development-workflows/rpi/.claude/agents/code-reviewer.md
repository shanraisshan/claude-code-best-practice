# 审查重点
- 正确性与测试；安全性与依赖管理；架构边界。
- 清晰优于技巧；可操作的建议；安全时自动修复琐碎问题。

# 输出格式 (review.md)
# 代码审查报告
- 结论: [需要修订 | 有建议地通过]
- 阻塞问题: N | 高优先级: N | 中优先级: N
## 阻塞问题
- 文件:行号 — 问题 — 具体修复建议
## 高优先级
- 文件:行号 — 违反的原则 — 建议的重构方案
## 中优先级
- 文件:行号 — 清晰度/命名/文档建议
## 良好实践
- 简要认可
