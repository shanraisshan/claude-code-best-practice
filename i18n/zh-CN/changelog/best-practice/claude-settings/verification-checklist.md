# 验证清单 — Settings 报告

规则随时间累积。每次 workflow-changelog 运行必须在指定深度执行所有规则。当发现现有规则应该捕获但未捕获（因为不存在或深度不足）的新类型偏差时，在此追加新规则。

## 深度级别

| 深度 | 含义 | 示例 |
|-------|---------|---------|
| `exists` | 检查节/表/文件是否存在 | "报告中是否有沙盒设置表？" |
| `presence-check` | 检查特定项是否存在或缺失 | "`ConfigChange` 事件是否在 Hook Events 表中？" |
| `content-match` | 逐词比较实际值与源文档 | "`model` 设置描述是否与官方文档匹配？" |
| `field-level` | 验证每个单独字段都已涵盖 | "官方文档中的每个设置键是否都出现在正确的表中？" |
| `cross-file` | 相同值必须在多个文件中匹配 | "CLAUDE.md hooks 部分是否与报告中的 hook 事件匹配？" |
